\section{目的と対象}

本書は、Telescope Array Low Energy Extension Surface Detector（TALE-SD、TALE 地表検出器）のモンテカルロ事象を対象とした、波形グラフニューラルネットワーク（Graph Neural Network: GNN）の現行仕様をまとめる。
対象は、幾何とエネルギーの再構成、および陽子・鉄の二値分類である。
本書の目的は、実装を逐一読まなくても、入力、グラフの作り方、ニューラルネットワークの層構成、出力、損失関数、学習設定を再現できるようにすることである。

現行の実行入口は次の 2 つである。
\begin{itemize}
  \item 再構成: \code{scripts/submit\_server\_waveform\_full\_training.sh}
  \item 質量分類のみ: \code{scripts/submit\_server\_mass\_only\_training.sh}
\end{itemize}
どちらのスクリプトも、Hierarchical Data Format version 5（HDF5）形式で保存済みのグラフ片を読み込む。
学習中に Data Summary Tape（DST）ファイルは読まない。
本書で比較に用いたグラフ入力は次である。
\begin{flushleft}
\small\code{/dicos\_ui\_home/ikomae/work/gnn/graphs/waveform\_gnn\_stream\_20260520\_123802}
\end{flushleft}
DST から作り直した energy-flat HDF5 が正常に完了したため、今後の学習では次のグラフ入力を標準とする。
\begin{flushleft}
\small\code{/dicos\_ui\_home/ikomae/work/gnn/graphs/}\\
\small\code{server\_graph\_export\_energyflat200000\_20260524\_075508/}\\
\small\code{server\_graph\_export\_energyflat200000\_20260524\_075508.h5}
\end{flushleft}
ただし、この最大規模 HDF5 は最終確認用であり、通常のパラメータ確認には重い。
現在のチューニングでは、旧基準データセットと近い容量を持つ \code{flat50000} を主に使う。
1 回の学習に半日から 1 日程度を要するため、数十回から数百回の再学習を前提にした特徴量選択は行わない。

\section{用語と表記}

本書では、略語を初めて用いる箇所で正式名称を示す。
その後は、読みやすさを保つために略語を用いる。
広く使われている日本語の用語がある場合は、日本語を主に用い、必要に応じて括弧内に英語名と略語を示す。

本書で用いる主な機械学習用語は次の意味である。

\begin{itemize}
  \item 特徴量とは、モデルに入力する数値である。本解析では、検出器の位置、信号時刻、信号量、波形、検出器間の時刻差などが特徴量に相当する。
  \item 教師ラベルとは、教師あり学習でモデルに与える正解値である。再構成では真のエネルギー、コア位置、到来方向を用い、質量分類では陽子または鉄のラベルを用いる。
  \item 標準化とは、学習データの平均を引き、標準偏差で割る変換である。特徴量のスケールをそろえることで、最適化を安定させる。
  \item バッチとは、1 回のパラメータ更新に使う事象の集合である。エポックとは、学習データ全体を 1 回使う単位である。
  \item 損失関数とは、予測と正解のずれを数値化する関数である。学習では、この値が小さくなるようにモデルのパラメータを更新する。ただし、負の対数尤度を含む場合など、損失の最小値が 0 である必要はない。
  \item SmoothL1 損失とは、小さい誤差では二乗誤差、大きい誤差では絶対値誤差として振る舞う損失である。しきい値を $\beta$、誤差を $r$ とすると、
  \[
    \mathrm{SmoothL1}_{\beta}(r)=
    \begin{cases}
      r^2/(2\beta), & |r|<\beta,\\
      |r|-\beta/2, & |r|\ge\beta
    \end{cases}
  \]
  である。外れ値の影響を二乗誤差より抑えつつ、誤差が小さい領域では滑らかに最適化できる。
  \item シグモイド関数とは、実数 $z$ を 0 から 1 の範囲に写す関数
  \[
    \sigma(z)=\frac{1}{1+\exp(-z)}
  \]
  である。関数形は S 字型であり、$\sigma(0)=0.5$、$z\rightarrow+\infty$ で $\sigma(z)\rightarrow1$、$z\rightarrow-\infty$ で $\sigma(z)\rightarrow0$ となる。
  二値分類では、$\sigma(z)$ を正例の確率として扱う。
  \item ロジット（logit）とは、確率 $p$ を実数に写す変換
  \[
    \mathrm{logit}(p)=\log\frac{p}{1-p}
  \]
  である。定義域は $0<p<1$ であり、$\mathrm{logit}(0.5)=0$、$p\rightarrow1$ で $+\infty$、$p\rightarrow0$ で $-\infty$ へ発散する。
  数学的には、logit は sigmoid の逆関数である。
  すなわち、
  \[
    \mathrm{logit}(\sigma(z))=z,\qquad
    \sigma(\mathrm{logit}(p))=p
  \]
  が成り立つ。
  ニューラルネットワークでは、確率に logit 関数を適用した値だけでなく、シグモイド関数に入れる前の実数出力 $z$ そのものもロジットと呼ぶ。
  本解析で「ロジット」と書く場合、通常はモデルが直接出力する生の値 $z$ を指す。
  確率として解釈するときだけ $\sigma(z)$ を計算する。
  質量分類では、$z$ が大きいほど鉄らしく、$z=0$ が $p(\mathrm{Fe})=0.5$ に対応する。
  \item 多層パーセプトロン（Multi-Layer Perceptron: MLP）とは、全結合層と非線形活性化関数を重ねたニューラルネットワークである。
  \item 畳み込みニューラルネットワーク（Convolutional Neural Network: CNN）とは、近接する入力点を共有重みで処理するニューラルネットワークである。本モデルでは、検出器平面ではなく、各パルスノードに付いた 1 次元波形に対して CNN を用いる。
  \item ゲート付き再帰ユニット（Gated Recurrent Unit: GRU）とは、時系列を順に処理する再帰型ニューラルネットワークである。波形の時間構造を要約するために用いる。
  \item プーリングとは、複数のベクトルを 1 つのベクトルに要約する処理である。本モデルでは、平均プーリング、最大プーリング、注意機構付きプーリングを用いる。
  \item 注意機構（attention）とは、入力要素ごとに重みを学習し、重要な要素を強く反映する仕組みである。
  \item ドロップアウトとは、学習中に一部の中間成分を確率的に無効化する正則化である。過学習を抑える目的で用いる。
  \item 層正規化（layer normalization: LayerNorm）とは、層の出力を正規化して学習を安定させる処理である。
  \item softplus 関数とは、$\log(1+\exp x)$ で定義される滑らかな正値化関数である。予想誤差のように負になってはいけない量を出力するときに用いる。
  \item 負の対数尤度（Negative Log-Likelihood: NLL）とは、確率モデルの尤もらしさを損失として表す量である。比較実験では、事象ごとの再構成誤差の大きさを同時に学習するために用いた。
  \item 二値交差エントロピー（Binary Cross Entropy: BCE）とは、二値分類で用いる標準的な損失関数である。本モデルでは陽子と鉄の分類、および品質スコアの学習に用いる。
  ロジット $z$ と教師値 $y\in[0,1]$ に対するロジット付き BCE は
  \[
    \mathrm{BCEWithLogits}(z,y)
    =
    -y\log\sigma(z)-(1-y)\log(1-\sigma(z))
  \]
  である。
  ここで入力は確率 $p$ ではなく、生のロジット $z$ である。
  実装では数値的に安定な形で、シグモイドと BCE をまとめて計算する。
  \item Root Mean Square Error（RMSE）とは、二乗平均平方根誤差である。$N$ 個の誤差 $r_i$ に対して
  \[
    \mathrm{RMSE}=\sqrt{\frac{1}{N}\sum_i r_i^2}
  \]
  と定義する。
  \item 中央 68\% 幅とは、誤差分布の 16 パーセンタイルから 84 パーセンタイルまでの幅である。半幅はその半分である。正規分布なら 1 標準偏差に対応するが、本解析では分布の形を仮定しないロバストな分解能指標として使う。
\end{itemize}

\section{事象グラフ}

1 つの空気シャワー事象を、1 つの有向グラフとして表す。
ここでいうグラフは、検出器を規則格子に並べた画像ではない。
Ising 型ノイズフィルタとは、パルス候補の採用・非採用を二値変数として扱い、時空間的に整合しないノイズ候補を落とす前処理である。
この前処理で残ったパルス候補を accepted pulse（採用パルス）と呼ぶ。
本モデルでは、accepted pulse をノードとし、検出器間の空間的・時間的な関係をエッジとする疎な不規則グラフを用いる。
同じ検出器に複数の accepted pulse が残る場合、その検出器は複数のノードを持つ。
ただし、パルスノード数が 4 未満、または異なる検出器数が 4 未満の事象は学習に使わない。

1 事象の HDF5 グループには、次のテンソルを保存する。
\[
  \mathbf{X}\in\R^{N_v\times27},\quad
  \mathbf{A}\in\R^{N_e\times13},\quad
  \mathbf{P}_{\mathrm{raw}}\in\R^{N_p\times7},\quad
  \mathbf{W}\in\R^{N_v\times4\times128}.
\]
ここで、$N_v$ は accepted-pulse ノード数、$N_e$ は有向エッジ数である。
$\mathbf{P}_{\mathrm{raw}}$ の第 1 列は、対応するノード番号である。
残りの 6 列が、パルス特徴量としてパルスエンコーダに入力される。

\subsection{ノード特徴量}

ノード特徴量は 27 個のスカラーで構成する。
これらは、検出器の位置、局所的な検出器間隔、事象内での相対位置、時刻、信号量、波形と較正に関係する量、およびパルス順序を表す。
列の順序は \code{NODE\_FEATURE\_COLUMNS} で固定されており、次の通りである。
\[
\begin{gathered}
x,y,z,\ d_{\mathrm{nearest}},\ \overline{d}_{3},\ N_{\mathrm{neigh}},\rho_{\mathrm{local}},
\Delta x,\Delta y,\Delta z,r,\\
t_{\mathrm{first}},t_{\mathrm{trig}},\log_{10}\rho_{\mathrm{first}},\sqrt{\rho_{\mathrm{first}}},
\log_{10}\rho_{\max},\\
N_{\mathrm{pulse}},\Delta t_{\mathrm{pulse}},N_{\mathrm{segment}},T_{\mathrm{wf}},
\log_{10}\mathrm{FADC}_{\max},
p_u,p_l,\sigma_u,\sigma_l,
\mathrm{pulse\ order},\mathrm{first\ pulse\ flag}.
\end{gathered}
\]
距離の単位は km、時刻の単位は $\mu\mathrm{s}$ である。
信号量は上下層の Flash Analog-to-Digital Converter（FADC）波形から求める。
また、信号量の一部は Vertical Equivalent Muon（VEM）相当の単位で較正される。

現行の標準設定では、任意の検出器番号埋め込みを用いない。
すなわち、\code{DETECTOR\_EMBEDDING\_DIM=0} である。
検出器ごとの意味の違いは、検出器番号そのものではなく、物理座標、局所的な検出器間隔、較正値、ノイズ量、波形応答、グラフの接続関係から学習させる。

表~\ref{tab:node-feature-definitions} に、各ノード特徴量の定義と採用理由を示す。
ここで「必要性」とは、物理的に入れる理由があるかどうかを示すものであり、最終的な有効性を保証するものではない。
各特徴量群の本当の必要性は、まず学習済みモデルに対する事後解析で確認する。
その上で、性能と説明性の両方に影響する候補だけを、少数の再学習で最終確認する。
波形エンコーダを導入した後は、波形から読める量と手作り特徴量の重複を明示的に調べる必要がある。

この表から分かるように、すべての特徴量が同じ意味で「必須」ではない。
$x,y,z$、相対位置、時刻、信号量は、再構成に直接関係するため優先度が高い。
一方、局所密度、pedestal、パルス数、波形長などは補助情報であり、低信号事象、低 multiplicity 事象、質量分類での寄与を個別に評価する。
以前の HDF5 には \code{log10\_total\_rho} と \code{sqrt\_total\_rho} という冗長列が含まれていたが、現行の入力からは削除した。
既存 HDF5 を読む場合も、この 2 列は読み込み時に落とし、モデルには渡さない。

特徴量の必要性は、まず学習済みチェックポイントに対する post-hoc 解析で調べる。
具体的には、特徴量を物理的に関連する group に分け、検証データまたはテストデータで group ablation または permutation を行い、再構成指標の悪化量を測る。
重要度は
\[
  I_g = M(\mathrm{perturb}(g))-M(\mathrm{original})
\]
で定義する。
ここで $g$ は特徴量 group、$M$ は \code{rmse\_log10\_energy}、\code{core\_68\_km}、\code{angular\_68\_deg} などの評価指標である。
この方法は再学習を必要としないため、\code{flat50000} のように 1 回の学習が半日から 1 日かかる条件でも実行できる。

最初に評価する group は次の通りである。
\begin{itemize}
  \item signal size group: $\rho$、$\log_{10}\rho$、$\sqrt{\rho}$、波形積分量、波形ピーク。
  \item timing group: ノード到来時刻、トリガー時刻、エッジの $\Delta t$、$|\Delta t|$、$\Delta t/d$。
  \item geometry group: 絶対位置、重心相対位置、エッジ距離。
  \item graph structure group: ノード数、エッジ数、近傍数、局所密度。
  \item waveform shape group: 上下層、raw/accepted、時間窓の形状。
  \item calibration and noise group: pedestal、pedestal sigma、FADC peak。
\end{itemize}

相関の強い特徴量は、個別に独立寄与を主張しない。
例えば \code{log10\_first\_rho} と \code{sqrt\_first\_rho} は同じ信号量の異なる表現であり、まず signal size group として評価する。
group が効くことを確認した後で、group 内の冗長性を conditional permutation または少数の reduced-feature 再学習で確認する。
性能が変わらず説明性が上がる場合は削除し、性能が落ちる場合、または有限データ・有限モデルで安定化に寄与する場合は残す。

\subsection{パルス特徴量}

第 1 列は対応するノード番号であり、これはパルスをノードへ集約するための index である。

現行実装では、1 つの accepted-pulse ノードに 1 行のパルス特徴量が対応する。

\subsection{波形特徴量}

各 accepted-pulse ノードには、長さ 128 bin の 4 チャネル波形を付ける。
\[
\mathbf{W}_v =
\left(
W^{u}_{\mathrm{raw}},
W^{l}_{\mathrm{raw}},
W^{u}_{\mathrm{accepted}},
W^{l}_{\mathrm{accepted}}
\right)\in\R^{4\times128}.
\]
$u$ は上層、$l$ は下層を表す。
1 bin は $0.02\,\mu\mathrm{s}$ であり、128 bin は $2.56\,\mu\mathrm{s}$ に対応する。
波形値は pedestal を引いた後、VEM 相当の単位へ較正し、極端な値はクリップして保存する。
4 チャネルの意味は表~\ref{tab:waveform-channel-definitions} の通りである。

\begin{table}[htbp]
  \centering
  \small
  \caption{波形 4 チャネルの定義と役割。}
  \label{tab:waveform-channel-definitions}
  \begin{tabular}{p{28mm}p{58mm}p{58mm}}
    \toprule
    チャネル & 定義 & 役割 \\
    \midrule
    $W^u_{\mathrm{raw}}$ &
    上層 FADC 波形を、対象 accepted pulse の立ち上がりで切り出した窓。 &
    上層の局所的な立ち上がり、幅、尾、基線を表す。 \\
    $W^l_{\mathrm{raw}}$ &
    下層 FADC 波形を、対象 accepted pulse の立ち上がりで切り出した窓。 &
    下層の局所的な立ち上がり、幅、尾、基線を表す。 \\
    $W^u_{\mathrm{accepted}}$ &
    上層波形のうち、accepted pulse 区間だけを残し、他の bin を 0 にした波形。 &

    下層波形のうち、accepted pulse 区間だけを残し、他の bin を 0 にした波形。 &

raw チャネルは、accepted pulse の立ち上がりを基準に切り出した較正済み波形窓である。
基準 bin は、上層と下層で検出された立ち上がり bin のうち早い方、
\[
  b_{\mathrm{rise}}=\min(b^{u}_{\mathrm{rise}},b^{l}_{\mathrm{rise}})
\]
を指す。
実装では、この bin の 32 bin 前から長さ 128 bin を切り出す。
したがって、波形の範囲内に十分な pre-pulse 領域がある場合、$b_{\mathrm{rise}}$ は出力窓の配列添字 32 の bin に対応する。
元の波形の先頭または末尾を越える部分は 0 で埋める。
この raw チャネルは、各 accepted pulse ノードごとに作られる。
この基準は、検出器間の絶対到達時刻をそろえるためではない。
検出器間の時刻差は、$t_{\mathrm{first}}$ やエッジの $\Delta t$ として別に入力する。
raw 波形窓では、各ノード内のパルス立ち上がり前後の形状を比較しやすくすることを目的として、立ち上がり位置を固定する。
したがって raw チャネルは、同じ検出器に複数の accepted pulse がある場合でも、対象ノードごとに異なる窓になる。

accepted-gapped チャネルは、Ising 型ノイズフィルタ後に残った accepted pulse 区間だけを値として残し、それ以外の bin を 0 にした波形である。
このチャネルでは、最も早い accepted pulse の立ち上がり bin を出力窓の先頭に置く。
後続パルスは、元波形上での bin 間隔を保って配置する。
したがって、パルスが存在しない区間は 0 で埋まり、パルス間の相対時刻差は bin 間隔として保存される。
128 bin を超える後続部分は切り捨てる。
同じ検出器に複数の accepted pulse ノードがある場合、accepted-gapped チャネルはその検出器の accepted pulse 列全体を表すため、これらのノードで同じ内容になる。
その一方で、raw チャネルは各ノードの対象パルスを中心に見るため、局所形状を分けて表す。
この設計により、raw チャネルは各ノードの局所的なパルス形状と前後の基線を表し、accepted-gapped チャネルはノイズ除去後のパルス列とパルス間隔を表す。
波形値はグラフ作成時に較正とクリップを行うが、学習時の標準化スケーラでは標準化しない。
この波形定義は HDF5 属性 \code{waveform\_schema=rise\_aligned\_raw\_plus\_accepted\_gapped\_v1} として保存する。
古い accepted-pulse compact 波形を含む HDF5 は、この定義と物理的に等価ではないため、学習時にエラーとして扱う。

\subsection{エッジ特徴量}

エッジは、異なる検出器 ID を持つノード間にだけ張る。
検出器間距離が $1.5\,\mathrm{km}$ 以下であり、かつ時刻差が $|\Delta t|\le 8\,\mu\mathrm{s}$ の組を候補とする。
採用した無向の組は、2 本の有向エッジとして保存する。
エッジ特徴量は 13 列であり、次の量から成る。
\[
(\Delta x,\Delta y,\Delta z,d,\Delta t,|\Delta t|,\Delta t/d,
\Delta\log_{10}\rho,
w_{\mathrm{ising}},w_{\mathrm{raw}},t_{\mathrm{excess}},s_{\mathrm{space}},s_{\mathrm{causal}}).
\]
ここで、エッジ $i\rightarrow j$ では
\[
  \Delta \mathbf{x}_{ij}=\mathbf{x}_j-\mathbf{x}_i,\qquad
  \Delta t_{ij}=t_j-t_i,\qquad
  \Delta\log_{10}\rho_{ij}=\log_{10}\rho_j-\log_{10}\rho_i
\]
である。
反対向きのエッジ $j\rightarrow i$ では、符号付きの量は符号が反転する。
距離 $d$、$|\Delta t|$、$t_{\mathrm{excess}}$、$s_{\mathrm{space}}$、$s_{\mathrm{causal}}$ は向きによらない。
ここで $s$ は score の意味で用いており、学習対象のパラメータではない。
具体的には
\[
  s_{\mathrm{space}}=\exp(-d/\lambda_{\mathrm{space}}),\qquad
  s_{\mathrm{causal}}=\exp(-t_{\mathrm{excess}}/\tau_{\mathrm{causal}})
\]
であり、現行実装では $\lambda_{\mathrm{space}}=0.8\,\mathrm{km}$、$\tau_{\mathrm{causal}}=1.0\,\mu\mathrm{s}$ を用いる。
$t_{\mathrm{excess}}$ は、光速伝播時間に $0.3\,\mu\mathrm{s}$ の許容幅を加えた後に残る余剰時刻差である。
各列の意味を表~\ref{tab:edge-feature-definitions} にまとめる。

$w_{\mathrm{raw}}$ は
\[
  w_{\mathrm{raw}}
  =
  s_{\mathrm{space}}s_{\mathrm{causal}}
  \left[
    1+0.12\sqrt{\log(1+\rho_i)\log(1+\rho_j)}
  \right]
\]
で定義する。
$w_{\mathrm{ising}}$ は、正の接続次数 $D_i,D_j$ を用いて
\[
  w_{\mathrm{ising}} =
  \frac{w_{\mathrm{raw}}}{(D_iD_j)^{0.35}}
\]
とする。
同じ検出器内の複数パルス間にはエッジを張らない。
検出器内のパルス順序と時間幅は、パルス特徴量と accepted-gapped 波形で表す。
シャワー面の幾何を決めるうえで、検出器間の時刻差は重要である。
そのため、現行モデルでは $\Delta t$、$|\Delta t|$、$\Delta t/d$ を通常のエッジ特徴量として使うだけでなく、専用の時刻差エンコーダにも入力する。

\subsection{教師ラベル}

再構成の教師ラベルは
\[
\mathbf{y}_{\mathrm{reco}}=
(\log_{10}E,\ x_c,y_c,z_c,\ n_x,n_y,n_z)\in\R^7
\]
である。
$E$ は eV 単位のエネルギー、$(x_c,y_c,z_c)$ は km 単位のコア位置、$(n_x,n_y,n_z)$ はシャワー到来方向の単位ベクトルである。

質量分類の教師ラベルは、\code{rusdmc} に記録された CORSIKA（COsmic Ray SImulations for KAscade）粒子番号から作る。
陽子は粒子番号 14 であり、ラベル 0 とする。
鉄は粒子番号 5626 であり、ラベル 1 とする。

\section{前処理とデータ分割}

ノード特徴量、エッジ特徴量、パルス特徴量、再構成ターゲットには、学習データだけから求めた標準化を適用する。
標準化は
\[
  x'=(x-\mu_{\mathrm{train}})/\sigma_{\mathrm{train}}
\]
で定義する。
ここで、$\mu_{\mathrm{train}}$ と $\sigma_{\mathrm{train}}$ は学習データにおける平均と標準偏差である。
標準偏差が数値的に 0 に近い列では、標準偏差を 1 として扱う。

モデルは標準化後のターゲットを出力する。
ただし、物理量に基づく損失関数と性能評価では、予測値と教師ラベルを物理単位に戻してから、エネルギー誤差、コア位置誤差、角度誤差を計算する。
この処理により、学習の数値安定性と、物理的に解釈しやすい損失の両方を保つ。

標準のデータ分割は \code{source-stratified} である。
これは、同じソースパスに由来する事象が異なる分割に混ざりにくいようにしながら、物理パラメータの分布をできるだけ保つ分割である。
分割は、ソースパス単位で行う。
ただし、比率はソースパス数だけではなく、各ソースパスに含まれるグラフ数に近づくように決める。
これは、1 つの DST ファイルに含まれる再構成可能事象数が異なる場合に、学習、検証、テストの実イベント数が大きく崩れることを避けるためである。
標準設定では、検証データを $5\%$、テストデータを $10\%$ とする。
残りの約 $85\%$ を学習に使う。
この比率は、DST を読み直してデータ数を増やす前提で、評価に十分な事象数を残しつつ、学習データを増やすための設定である。
学習時には、各エポックの開始時に学習インデックス全体をシャッフルしてからバッチを作る。
検証データとテストデータの順序は固定する。
この設計は、HDF5 内のエネルギー順、ソース順、粒子種順がバッチ内に残ることを防ぐためのものである。

\subsection{ノードごとのエンコーダ}

パルス特徴量は、ノード番号の列を除いた 6 次元の入力である。
この入力を
\[
  \mathrm{MLP}_{\mathrm{pulse}}:\R^6\rightarrow\R^{192}\rightarrow\R^{192}
\]
で符号化する。
活性化関数には Sigmoid Linear Unit（SiLU）を用い、過学習を抑えるためにドロップアウトを加える。
同じノードに対応するパルス表現は、平均プーリングと最大プーリングで集約する。
その結果、各ノードにつき $2H=384$ 次元のパルス表現が得られる。

波形エンコーダは、まず次の 1 次元畳み込みを適用する。
\[
\mathrm{Conv1D}(4\rightarrow32,k=5)\rightarrow\mathrm{SiLU}
\rightarrow\mathrm{LayerNorm}
\rightarrow\mathrm{Conv1D}(32\rightarrow64,k=5)\rightarrow\mathrm{SiLU}.
\]
標準の \code{cnn-gru} モードでは、その後に隠れ次元 32 の双方向 GRU を適用する。
GRU の最終状態、CNN 系列の平均、CNN 系列の最大値を連結し、64 次元の波形埋め込みに射影する。

ノードエンコーダは、ノード特徴量、波形埋め込み、パルス表現を連結した入力を受け取る。
標準設定での入力次元は
\[
  27\ \oplus\ 64\ \oplus\ 384 = 475
\]
である。
この 475 次元の入力を
\[
  \R^{475}\rightarrow\R^{192}\rightarrow\R^{192}
\]
の MLP で 192 次元のノード状態に変換する。
各線形層の後には SiLU と LayerNorm を置く。

\subsection{時刻差エンコーダとメッセージ伝播}

メッセージ伝播の前に、時刻差エンコーダがエッジの時刻差特徴量
$(\Delta t,|\Delta t|,\Delta t/d)$
を MLP で処理する。
その出力を、到着先ノードごとに平均してノード状態に加え、LayerNorm で正規化する。
この処理は、シャワー面の到達時刻差を通常のエッジ特徴量より明示的に使わせるためのものである。

5 層の GNN は、ゲート付きエッジメッセージ伝播層で構成される。
有向エッジ $i\rightarrow j$ のエッジ特徴量を $a_{ij}$ とすると、メッセージの入力は
\[
  [h_i,h_j,a_{ij}]\in\R^{2H+13}=\R^{397}
\]
である。
メッセージ MLP はこの入力を 192 次元に写す。
別のゲート MLP はスカラーのシグモイド値を出力する。
したがって、有効なメッセージは
\[
  m_{ij}=\mathrm{MLP}_m([h_i,h_j,a_{ij}])\,
  \sigma(\mathrm{MLP}_g([h_i,h_j,a_{ij}]))
\]
である。

各ノードでは、入力してくるメッセージの平均と最大値を計算する。
ノード更新では、現在のノード状態、平均メッセージ、最大メッセージを連結した
$[h_j,\overline{m}_j,m^{\max}_j]\in\R^{3H}$
を用いる。
更新後には、残差接続、LayerNorm、フィードフォワードネットワーク（feed-forward network: FFN）を適用する。
残差接続は、入力を出力に足し戻す処理であり、深いネットワークの学習を安定させる。

\subsection{グラフ読み出し}

メッセージ伝播後のノード状態を、グラフ全体の 1 つの事象ベクトルに変換する。
この処理をグラフ読み出しと呼ぶ。
本モデルでは、全ノードの平均、全ノードの最大値、4 個の注意機構付き読み出しヘッドを連結する。
したがって、グラフ読み出しの次元は
\[
  H(2+K)=192\times6=1152
\]
である。
再構成では、この 1152 次元ベクトルを共有 MLP
\[
  \R^{1152}\rightarrow\R^{384}\rightarrow\R^{192}
\]
に通す。
得られた 192 次元の事象表現を、再構成出力と品質出力に入力する。

\section{再構成モデル}

再構成用スクリプトは、エネルギー、コア位置、到来方向、および事象品質を学習する。
標準コマンドは
\[
\code{scripts/submit\_server\_waveform\_full\_training.sh}
\]
である。
2026 年 5 月 24 日時点の標準設定では、\code{TRAINING\_TASK=reconstruction}、\code{LOSS\_MODE=physics}、\code{QUALITY\_PREDICTION=1}、\code{ERROR\_PREDICTION=0} とする。
現行の標準最大エポック数は 128 である。
検証損失が 12 エポック改善しなければ学習を停止する。
ただし、検証損失が一時的に悪化してから再び改善する場合を考慮し、早期終了は 32 エポック目までは有効にしない。
検証損失が最小になった checkpoint は学習中に保存し、最終的な validation/test 評価では、その best validation checkpoint を読み戻して使う。

\subsection{出力}

再構成出力は、標準化された 7 つのターゲット座標である。
\[
  \hat{\mathbf{y}} =
  (\widehat{\log_{10}E},\hat{x}_c,\hat{y}_c,\hat{z}_c,\hat{n}_x,\hat{n}_y,\hat{n}_z).
\]
エネルギー出力層は、共有事象表現から 1 次元を出力する線形層である。
コア出力層と方向出力層は、同じ共有事象表現を入力とする 2 層の MLP である。
方向は 3 成分のベクトルとして出力する。
角度誤差を計算するときは、予測ベクトルと真のベクトルを内部で正規化してから角距離を求める。

品質出力層は 1 つのロジットを出力する。
ロジットとは、用語節で定義した通り、シグモイド関数に入れる前の実数値である。
シグモイド関数を適用した値を、0 から 1 までの事象品質スコアとして解釈する。
この品質スコアは、再構成結果を物理解析に使うときの事象選択に用いる。

\subsection{物理再構成損失}

物理再構成損失は、標準化された予測値と教師ラベルを物理単位に戻してから計算する。
標準の重みは
\[
  w_E=1.2,\quad w_c=1.0,\quad w_\theta=1.4
\]
であり、損失は
\[
  L_{\mathrm{phys}} =
  w_E L_E + w_c L_c + w_\theta L_\theta
\]
である。

エネルギー項は、$\log_{10}E$ の差に対する SmoothL1 損失である。
SmoothL1 損失は、用語節で定義した通り、誤差が小さい領域では二乗誤差に近く、誤差が大きい領域では絶対値誤差に近い。
外れ値の影響を抑えながら、中心付近の精度を上げるために用いる。
\[
  L_E=\mathrm{SmoothL1}(\widehat{\log_{10}E}-\log_{10}E;\beta=0.05).
\]

コア項は、水平方向のコア誤差を $0.12\,\mathrm{km}$ で規格化した二乗誤差である。
\[
  L_c=\left\langle
  \frac{(\hat{x}_c-x_c)^2+(\hat{y}_c-y_c)^2}{(0.12\,\mathrm{km})^2}
  \right\rangle.
\]

方向項は、予測方向と真の方向の角距離 $\alpha$ を $1.0^\circ$ で規格化し、SmoothL1 損失を適用する。
\[
  L_\theta=\mathrm{SmoothL1}\left(\frac{\alpha}{1.0^\circ};\beta=1\right).
\]
この規格化により、角度誤差が損失全体の中で過小評価されることを防ぐ。

\subsection{品質損失}

品質教師値は、現在の予測から計算した残差を用いて作る。
ただし、この教師値は計算グラフから切り離す。
したがって、品質出力は再構成出力を直接改善するためではなく、「この事象の再構成がどの程度信用できるか」を学習するために用いる。
\[
  q_{\mathrm{target}}=
  \exp\left[-\frac{1}{3}\left(
  \frac{|E_{\mathrm{rec}}/E_{\mathrm{true}}-1|}{0.10}
  +\frac{\Delta r_c}{0.05\,\mathrm{km}}
  +\frac{\alpha}{1.0^\circ}
  \right)\right].
\]
品質出力層は、ロジット付き二値交差エントロピーで学習する。
\[
  L_q=\mathrm{BCEWithLogits}(q_{\mathrm{logit}},q_{\mathrm{target}}),
  \qquad w_q=0.2.
\]
標準設定での再構成の全損失は
\[
  L_{\mathrm{total}}=
  L_{\mathrm{phys}}
  +0.2\,L_q
\]
である。
品質スコアは、解析に使う事象を選ぶための 1 つの総合指標として用いる。
例えば、品質スコア上位 $95\%$、$90\%$、$80\%$ を残した場合の精度と残存率を評価する。

\subsection{予想誤差出力と負の対数尤度の扱い}

実装には、事象ごとの予想誤差を出力する head と、ガウス分布の負の対数尤度を再構成損失に加える \code{physics-nll} モードがある。
この出力では、3 つの raw 値を softplus 関数で正の値に変換し、
\[
  (\sigma_E,\sigma_\theta,\sigma_c)
  =\mathrm{softplus}(r)\odot(0.10,\ 0.8^\circ,\ 0.04\,\mathrm{km})
\]
として、相対エネルギー誤差、角度誤差、コア位置誤差の予想値を得る。

負の対数尤度は、予測誤差 $\sigma$ を小さくしすぎると $r^2/\sigma^2$ が大きくなり、逆に大きくしすぎると $\log\sigma$ が大きくなる、という性質を使う。
固定された残差 $r$ に対して
\[
  \frac{1}{2}\frac{r^2}{\sigma^2}+\log\sigma
\]
を最小化すると、概念的には $\sigma^2\simeq r^2$ が好まれる。
しかし、2026 年 5 月 22--23 日の比較では、この方法を標準設定に入れる根拠は得られなかった。
そのため、現時点では \code{ERROR\_PREDICTION=0}、\code{NLL\_WEIGHT=0.0} を標準とする。
予想誤差 head を再び試す場合は、まず再構成損失に混ぜず、実際の誤差を教師値とする補助損失として別に評価する必要がある。

\section{質量分類のみのモデル}

質量分類のみの学習には
\[
\code{scripts/submit\_server\_mass\_only\_training.sh}
\]
を用いる。
このスクリプトは、同じ学習ラッパーを呼び出し、\code{TRAINING\_TASK=mass}、\code{MASS\_CLASSIFICATION=1}、\code{QUALITY\_PREDICTION=0}、\code{ERROR\_PREDICTION=0} を設定する。
現行の標準最大エポック数は 128 である。
検証損失が 12 エポック改善しなければ停止し、32 エポック目までは早期終了しない。
再構成と同じく、最終評価では best validation checkpoint を読み戻す。
モデル定義上は再構成出力層も存在するが、質量分類のみの学習では再構成損失を使わない。

\subsection{質量分類ヘッド}

現行の質量分類では、拡張分類構造（enhanced classification architecture）を用いる。
これは、最終的な共有事象表現だけでなく、初期ノード表現や波形の直接要約も分類に使う構造である。
入力は次の 4 種類のベクトルを連結したものである。

\begin{itemize}
  \item 最終グラフ読み出し。次元は 1152 である。
  \item ノードエンコーダ直後の平均プーリングと最大プーリング。次元は $2H=384$ である。
  \item グラフの規模を表す文脈量 $(\log(1+N_v),\log(1+N_e),N_e/N_v)$。次元は 3 である。
  \item 波形の直接読み出し。波形埋め込みの平均と最大値、および波形形状要約の平均と最大値を連結する。次元は $2D_w+2\times20=168$ である。
\end{itemize}

したがって、分類ヘッドの入力次元は
\[
  1152+384+3+168=1707
\]
である。
分類ヘッドは
\[
\mathrm{LayerNorm}(1707)
\rightarrow384
\rightarrow192
\rightarrow\mathrm{LayerNorm}
\rightarrow192
\rightarrow1
\]
である。
各中間層には SiLU とドロップアウトを用いる。
出力は 1 つのロジット $z$ である。
シグモイド関数を適用した
\[
  p(\mathrm{Fe})=\sigma(z)
\]
を、鉄である確率として扱う。
ラベルの定義は、陽子が 0、鉄が 1 である。

\subsection{質量分類損失}

標準の質量分類損失は、ロジット付き二値交差エントロピーである。
\[
  L_{\mathrm{BCE}}=
  -y\log\sigma(z)-(1-y)\log(1-\sigma(z)).
\]
現行設定では、正例である鉄に対する追加のクラス重みは使わない。
すなわち、\code{MASS\_POS\_WEIGHT\_MODE=none} である。

さらに、同じバッチに陽子と鉄が含まれる場合にはランキング損失を加える。
鉄事象のロジットを $z^+$、陽子事象のロジットを $z^-$ とすると、
\[
  L_{\mathrm{rank}}=
  \left\langle
  \mathrm{softplus}\left(1.0-(z^+-z^-)\right)
  \right\rangle
\]
である。
この項は、鉄のロジットが陽子のロジットより大きくなるように促す。
バッチ内に片方の粒子種しか含まれない場合、このランキング項はそのバッチでは計算しない。

標準設定での質量分類の全損失は
\[
  L_{\mathrm{mass}} = L_{\mathrm{BCE}} + 0.5\,L_{\mathrm{rank}}
\]
である。
現行スクリプトでは、質量分類の過学習を抑えるために \code{DROPOUT=0.12}、\code{WEIGHT\_DECAY=5e-4}、\code{LR\_SCHEDULER=cosine} を標準にしている。
学習中の自然な分類境界は $p(\mathrm{Fe})=0.5$ である。
診断では、検証データを用いて分類しきい値を後から調整することもできる。
しかし、モデル自体は、ロジット 0 が陽子と鉄の境界になるように学習させる。

\subsection{現在の質量分類結果}

旧基準データセットを用いた \code{server\_mass\_waveform\_direct\_48epoch\_20260522\_143042} では、テスト accuracy 0.800、balanced accuracy 0.801、AUC 0.887 であった。
\code{flat20000} を用いた \code{server\_mass\_b6000\_128epoch\_20260527\_042231} では、テスト accuracy 0.790、balanced accuracy 0.791、AUC 0.875 であった。
後者の split は train 993,451、validation 58,487、test 117,082 であり、best validation epoch は 12 である。
この 2 つは入力 HDF5 と split が異なるため、モデルだけの比較ではない。
次の比較では、同じ HDF5 と同じ split 条件で、質量分類設定だけを固定して確認する。
